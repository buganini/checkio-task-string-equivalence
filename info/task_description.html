In Unicode, a glyph can be represented differently in binary form with only one way to check the equivalence.

Due to historical reasons, some glyphs were brought into Unicode multiple times with similar or different forms and with this, the problem of equivalence emerged.

Combining sequence Ç C+◌̧
Ordering of combining marks q + ◌̇ + ◌̣ q + ◌̣ + ◌̇
Hangul 가 ᄀ + ᅡ
Singleton Ω (U+03A9) Ω (U+2126)

Canonical equivalence cases defined in UAX #15. You should write a function that checks the equivalence for two strings.

Canonical equivalent glyphs must be mutually exchangeable under any circumstance; so the Cyrillic "A" is not equivalent to Latin "A".

Input: Two arguments as strings (unicode).

Output: Are they equivalent or not as a boolean.

Example: 

checkio(u"\u212B", u"\u00C5") == True
checkio(u"\u212B", u"A") == False
checkio("q\u0323\u0307", "q\u0307\u0323") == True

How it is used: If you are making a social platform and want to have unique user-names but without restricting users to Latin characters, this is a concept you should know about.
